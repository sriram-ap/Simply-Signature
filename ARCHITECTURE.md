# Architecture

## Design goals

1. **Zero-maintenance backend** — there isn't one. No database, no API, no auth, no cookies. The entire site is prerendered static HTML; the only "write" path is a WhatsApp deep link composed in the browser.
2. **Non-technical weekly updates** — everything that changes weekly lives in `config/*.json` and `public/assets/posters/`. The admin never touches TypeScript.
3. **Premium feel on a home-kitchen budget** — system of design tokens, serif/sans font pairing (Fraunces + Inter), restrained motion, and real brand assets (weekly posters) instead of stock photography.

## Data flow

```
config/menu.json ──┐
config/gallery.json ├─ imported at build time ─→ src/lib/menu.ts (typed) ─→ pages
config/reviews.json ┘

src/data/dishes.ts  ─→ /menu, /menu/[slug], home page cards, sitemap
src/data/faqs.ts    ─→ /faq (+ FAQPage JSON-LD)
src/lib/site.ts     ─→ layout metadata, footer, contact, Restaurant JSON-LD
```

- JSON is imported statically, so **a config edit is a code change**: pushing it triggers a Vercel build and every page that uses the value is re-rendered. There is no runtime fetch and no cache to invalidate.
- `src/lib/menu.ts` casts the JSON to typed interfaces (`WeeklyMenu`, `MenuItem`, …). A malformed edit fails the build (TypeScript + Next prerender), which is the safety net that keeps bad config from going live.

## Rendering model

Every route is static (`○`/`●` in the build output):

| Route | Source of truth |
| --- | --- |
| `/` | `menu.json` + `dishes.ts` + `site.ts` |
| `/specials`, `/order` | `menu.json` |
| `/menu`, `/menu/[slug]` (one per dish, `generateStaticParams`) | `dishes.ts` |
| `/gallery` | `gallery.json` |
| `/reviews` | `reviews.json` |
| `/faq` | `faqs.ts` |
| `/about`, `/contact`, `/_not-found` | `site.ts` |
| `robots.txt`, `sitemap.xml`, `manifest.webmanifest`, `opengraph-image` | generated by route handlers in `src/app/` |

The only client components are the ones that need interactivity:

- `order-form.tsx` — quantities, validation, WhatsApp URL + clipboard.
- `site-header.tsx` — mobile menu, active-link highlight.
- `copy-button.tsx` — clipboard with graceful fallback.
- `reveal.tsx` — scroll-reveal animation (skipped entirely under `prefers-reduced-motion`).

## Order flow

1. User picks quantities (per-item `maxQty` caps enforced in state).
2. Client-side validation: ≥1 item, name, 10-digit Indian mobile (`^[6-9]\d{9}$`), flat number. Errors are announced via `role="alert"` / `aria-live`.
3. `buildMessage()` composes a plain-text order (items, totals, customer details).
4. "Order on WhatsApp" opens `https://wa.me/<number>?text=<encoded>` — the user reviews and presses send themselves. Nothing is transmitted by the site and nothing is stored.
5. "Copy order message" copies the same text for posting anywhere (MyGate chat, SMS).

The weekly **MyGate announcement** (admin-authored in `menu.json → mygateMessage`) is surfaced on `/specials` with a copy button, so any neighbour can help spread the word.

## Styling system

Tailwind v4 with all brand tokens declared in `@theme` (`src/app/globals.css`):

- **Palette** — `evergreen-*` (deep greens from the brand posters), `cream-*`, `gold-*`, `terracotta-*`, `ink`.
- **Fonts** — `--font-display` (Fraunces, italicised for the "Simply" wordmark) and `--font-sans` (Inter), loaded via `next/font` (self-hosted, zero layout shift).
- **Shadows/motion** — `shadow-card`, `shadow-card-hover`, `animate-fade-up`.

shadcn/ui conventions are followed (cva variants + `cn()` merge) without the Radix dependency — the only composite widgets needed (accordion, radio cards) are built on native `<details>` and `<input type=radio>`, which keeps the bundle small and accessibility native.

## SEO

- Per-page `metadata` exports with canonical URLs; global template in `layout.tsx`.
- `Restaurant` JSON-LD (layout), `MenuItem` JSON-LD (dish pages), `FAQPage` JSON-LD (FAQ).
- `robots.ts`, `sitemap.ts` (every page + dish URL, derived from `dishes.ts`), `manifest.ts`, and a build-time generated Open Graph image (`opengraph-image.tsx`, 1200×630).

## Accessibility

- Skip link, semantic landmarks, one `h1` per page, labelled nav regions.
- Steppers are real buttons with `aria-label`s; live totals use `aria-live="polite"`.
- Focus-visible ring on brand gold; colour pairs chosen for WCAG AA on both cream and evergreen surfaces.
- All motion honours `prefers-reduced-motion` (component-level and CSS-level).

## CI/CD

- **GitHub Actions** (`.github/workflows/ci.yml`): typecheck + lint + build on every push/PR to `main`.
- **Vercel**: auto-deploys `main`; PRs get preview deployments. `vercel.json` adds security headers (nosniff, frame-deny, referrer policy) and immutable cache for `/assets/*`.
